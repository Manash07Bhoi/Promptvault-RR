## 2025-03-15 - ILIKE Wildcard Injection Prevention\n**Vulnerability:** Unsanitized user inputs passing directly into PostgreSQL `ILIKE` queries via Drizzle ORM (e.g., `ilike(col, \`%${q}%\`)`). This allows wildcard injection (`%`, `_`), which can lead to unintended data exposure or ReDoS.\n**Learning:** While Drizzle parameterizes queries against classic SQL injection, `LIKE`/`ILIKE` wildcards still execute their special meaning if not escaped.\n**Prevention:** Create a centralized utility `pattern.replace(/[%_\\\\]/g, "\\\\$&")` and wrap all dynamic inputs in `LIKE`/`ILIKE` conditions.
