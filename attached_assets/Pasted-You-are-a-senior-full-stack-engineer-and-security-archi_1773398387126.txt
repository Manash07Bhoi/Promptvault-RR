── 2.1 Authentication & JWT ──

□ JWT refresh tokens expire in exactly 30 days

□ Refresh token rotation: each use issues a new refresh token
  and invalidates the old one (stored in Redis or DB)
□ Refresh tokens stored in httpOnly + Secure + SameSite=Strict cookie
  Access tokens stored in memory only (never localStorage, never cookie)
□ All protected routes verify token on EVERY request — no exceptions
□ Token payload contains only: userId, role, sessionId — nothing else
□ JWT secret/keys are minimum 256 bits of entropy
□ Implement token blacklist for logout: store revoked JTIs in Redis
  with TTL matching remaining token lifetime
□ Verify: expired tokens return 401, tampered tokens return 401,
  missing tokens return 401 — never 500

── 2.2 Password Security ──
