□ Hardcoded user data: any component that renders a static

  data from the Zustand auth store or React Query.
  
  ❌ const user = { name: "John Doe", email: "test@example.com" }
  ✅ const { user } = useAuthStore()

□ Hardcoded pack data: any component that has static pack 
  titles, prices, descriptions, or download counts must 
  instead consume the /api/packs endpoints via React Query.
  
  ❌ const packs = [{ title: "100 Marketing Prompts", price: 19.99 }]
  ✅ const { data: packs } = useQuery({ queryKey: queryKeys.packs.featured(), ... })

  ✅ const { data } = useQuery({ queryKey: ['analytics', 'revenue'],

  grep -r "mock\|fake\|dummy\|hardcoded\|TODO\|FIXME\|lorem\|placeholder" \
    server/src/ --include="*.ts" -l

□ Any service function that returns a hardcoded array or object 
  when an API key is missing must instead return a clear error 
  response (not fake data) OR a documented fallback with proper 
  logging. Distinguish between: development fallback (acceptable

  use real data only).

PHASE 5 — LOGIC BUGS & EDGE CASE FIXES
═══════════════════════════════════════════════════════════════════

These are the non-obvious bugs that only appear under specific

── 5.1 Race Conditions ──

□ Double-click Buy button: ensure the checkout session is only 
  created once. The button must be disabled on first click. 
  Verify no duplicate orders in DB on rapid double-click.

□ Double-click Wishlist button: rapid toggle should not create 
  duplicate SavedPack rows. Use a debounce of 300ms on the 
  toggle handler.

□ Double-click "Approve" in admin: should not create duplicate 
  moderation log entries. Use idempotency check on the server: 
  if pack.status is already PUBLISHED, return 200 without 
  creating another log entry.

□ Simultaneous coupon redemption: two users applying the same 
  single-use coupon at the exact same time. Fix: use a DB 
  transaction with SELECT FOR UPDATE on the coupon row, 
  check usesCount, increment only if < maxUses, commit.

── 5.2 State Synchronisation Bugs ──

□ After purchasing a pack, the pack card on the explore page 
  should no longer show "Buy Now" — it should show "View Pack" 
  or "Download". Test: buy a pack → go back to /explore → 
  find that pack → verify the button state changed.
  Fix: invalidate React Query user purchases cache on checkout 
  success and add a usePurchasedPackIds() hook consumed by 
  pack cards.

□ After admin approves a pack, the approval queue count badge 
  in the sidebar must decrement immediately via WebSocket. 
  Test: have approval queue open, approve a pack, verify count 
  decrements without page refresh.

□ After editing a category name in admin, the category filter 
  in /explore must show the new name. Test: rename a category → 
  go to /explore → verify sidebar filter shows new name.
  Fix: categories are cached — ensure the cache invalidates 
  on category PATCH.

□ After logging out in one browser tab, other open tabs should 
  detect the logout (storage event or periodic /me polling) 
  and redirect to login. Implement: listen to a "logout" 
  BroadcastChannel event sent from the logout handler.

── 5.3 Pagination Edge Cases ──

□ Empty first page: if a filter produces zero results, the
