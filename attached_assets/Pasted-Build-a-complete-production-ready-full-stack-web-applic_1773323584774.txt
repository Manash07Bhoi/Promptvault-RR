Build a complete, production-ready full-stack web application called "PromptVault" — an AI Prompt Marketplace platform. Follow every instruction below exactly and completely before stopping.

═══════════════════════════════════════════════
TECH STACK — USE EXACTLY THESE
═══════════════════════════════════════════════

Frontend:
- React 18 + TypeScript + Vite
- Tailwind CSS (dark mode default)
- Framer Motion (animations)
- Zustand (state management)
- React Query / TanStack Query (server state)
- React Router v6 (routing)
- Recharts (admin charts)
- Lucide React (icons)
- Axios (HTTP client)

Backend:
- Node.js + Express + TypeScript
- PostgreSQL (via pg + Prisma ORM)
- Redis (via ioredis + BullMQ for job queues)
- JWT authentication (jsonwebtoken, RS256)
- bcrypt (password hashing)
- Multer + Sharp (image processing)
- Puppeteer (PDF generation)
- Archiver (ZIP generation)
- AWS SDK v3 (S3 file storage — use local disk as fallback if no S3 keys)
- Stripe (payments)
- Nodemailer / Resend (email)
- Socket.IO (WebSocket real-time events)
- Zod (request validation)
- Helmet + cors + express-rate-limit (security)

PHASE 2 — COMPLETE BACKEND
═══════════════════════════════════════════════

Build ALL of these in server/src/:

── MIDDLEWARE ──
- auth.middleware.ts — JWT verification, attach user to req
- admin.middleware.ts — Check role === ADMIN or SUPER_ADMIN
- validate.middleware.ts — Zod schema validation wrapper
- rateLimit.middleware.ts — Per-route rate limiters
- errorHandler.middleware.ts — Global error handler, consistent JSON error envelope

── SERVICES ──

1. ai.service.ts
   - generatePrompts(category, count, packTitle, aiTools) — calls Anthropic Claude API (model: claude-3-5-sonnet-20241022), returns array of {title, body, useCase, exampleOutput}
   - generatePackMetadata(category, prompts[]) — returns {title, description, shortDescription, tags, seoTitle, seoDescription, suggestedPriceCents}
   - If ANTHROPIC_API_KEY not set, return high-quality hardcoded mock data so the app still works

2. pdf.service.ts
   - generatePackPDF(pack, prompts[]) — uses Puppeteer to render a beautiful HTML template to PDF
   - HTML template: dark background, pack title header, numbered prompt list with styled code blocks, footer with promptvault.com
   - Save to /tmp/files/{packId}.pdf, return file path

3. zip.service.ts
   - generatePackZIP(pack, pdfPath, prompts[]) — creates ZIP with PDF + prompts.txt + prompts.json + README.md
   - Save to /tmp/files/{packId}.zip, return file path

4. storage.service.ts
   - uploadFile(localPath, key) — upload to S3 if AWS env vars set, else copy to /public/files/ and return local URL
   - getSignedUrl(key) — presigned S3 URL or direct URL for local

5. stripe.service.ts
   - createCheckoutSession(userId, packIds[], couponCode?) — Stripe checkout session
   - handleWebhook(event) — handle payment_intent.succeeded, charge.refunded
   - If STRIPE_SECRET_KEY not set, use mock checkout that immediately creates a completed order

6. email.service.ts
   - sendPurchaseConfirmation(user, order, packs[])
   - sendPasswordReset(email, token)
   - sendEmailVerification(email, token)
   - Uses Resend if RESEND_API_KEY set, else logs to console

7. image.service.ts
   - generatePackImage(packTitle, category, color) — calls DALL-E 3 API if key set
   - Falls back to generating a beautiful SVG thumbnail programmatically using Sharp

8. search.service.ts
   - indexPack(pack) — add/update pack in Meilisearch if configured
   - searchPacks(query, filters) — search via Meilisearch or fallback to PostgreSQL ILIKE

── WORKERS (BullMQ) ──

automation.worker.ts — processes jobs from "automation" queue:
- GENERATE_PROMPTS: call ai.service.generatePrompts, save to DB
- GENERATE_PACK: orchestrate full pipeline (prompts → metadata → image → PDF → ZIP → listing)
- GENERATE_PDF: call pdf.service, upload via storage.service, update GeneratedFile
- GENERATE_ZIP: call zip.service, upload, update GeneratedFile
- GENERATE_IMAGE: call image.service, upload, update pack thumbnailUrl
- CREATE_LISTING: set pack status to AI_GENERATED, create moderation queue entry, emit WebSocket event

Each job must:
- Update AutomationJob record with status/progress
- Handle errors gracefully with retry logic
- Log to console with job ID and type

Build full CRUD + business logic for ALL endpoints:
